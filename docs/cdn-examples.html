BP Animate - CDN Usage Examples

Different ways to include BP Animate in your projects

1. GitHub Raw Links (Auto-Update)

Links directly to the main branch. Updates automatically when you push changes.

<!-- CSS -->
<link rel="stylesheet" href="https://raw.githubusercontent.com/braudypedrosa/bp-animate/main/dist/bp-animate.css">

<!-- JavaScript -->
<script src="https://raw.githubusercontent.com/braudypedrosa/bp-animate/main/dist/bp-animate.js"></script>
Note: Raw GitHub links work but aren't ideal for production. Use jsDelivr instead for better performance.

2. jsDelivr CDN - Latest (Auto-Update)

Uses jsDelivr CDN pointing to main branch. Auto-updates with better caching.

<!-- CSS -->
<link rel="stylesheet" href="https://cdn.jsdelivr.net/gh/braudypedrosa/bp-animate@main/dist/bp-animate.css">

<!-- JavaScript -->
<script src="https://cdn.jsdelivr.net/gh/braudypedrosa/bp-animate@main/dist/bp-animate.js"></script>
Recommended for development: Auto-updates when you push to main branch.

3. jsDelivr CDN - Specific Version (Stable)

Pin to a specific version for production stability.

<!-- CSS -->
<link rel="stylesheet" href="https://cdn.jsdelivr.net/gh/braudypedrosa/bp-animate@1.1.1/dist/bp-animate.css">

<!-- JavaScript -->
<script src="https://cdn.jsdelivr.net/gh/braudypedrosa/bp-animate@1.1.1/dist/bp-animate.js"></script>
Recommended for production: Version pinning ensures stability. Update version number when needed.

4. jsDelivr CDN - Minified Version

Use minified CSS for smaller file size.

<!-- CSS (Minified) -->
<link rel="stylesheet" href="https://cdn.jsdelivr.net/gh/braudypedrosa/bp-animate@main/dist/bp-animate.min.css">

<!-- JavaScript -->
<script src="https://cdn.jsdelivr.net/gh/braudypedrosa/bp-animate@main/dist/bp-animate.js"></script>

5. Using GitHub Releases/Tags

Link to a specific release tag for version control.

<!-- After creating a release tag (e.g., v1.1.1) -->
<link rel="stylesheet" href="https://cdn.jsdelivr.net/gh/braudypedrosa/bp-animate@v1.1.1/dist/bp-animate.css">
<script src="https://cdn.jsdelivr.net/gh/braudypedrosa/bp-animate@v1.1.1/dist/bp-animate.js"></script>
Best practice: Create GitHub releases with tags (v1.1.1) for better version management.

Complete Example

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>My Page</title>
    
    <!-- BP Animate CSS -->
    <link rel="stylesheet" href="https://cdn.jsdelivr.net/gh/braudypedrosa/bp-animate@main/dist/bp-animate.css">
</head>
<body>
    <div class="bp-animate fade-in">
        <p>This will animate when scrolled into view</p>
    </div>
    
    <!-- BP Animate JavaScript -->
    <script src="https://cdn.jsdelivr.net/gh/braudypedrosa/bp-animate@main/dist/bp-animate.js"></script>
</body>
</html>

How to Update

For auto-updating links (@main):

  • Just push your changes to the main branch
  • jsDelivr will cache the new version (may take a few minutes)
  • Users will get the update automatically

For versioned links (@1.1.1):

  • Update the version number in your HTML when you want to upgrade
  • Or create a new release tag and update the link
